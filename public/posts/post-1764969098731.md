# 快速求平方根的倒数

快速平方根倒数算法（Fast Inverse Square Root），是一种在 IEEE 754 浮点格式下快速估计 $\frac{1}{\sqrt{x}}$ 的方法。

## 早期应用

该算法最出名的是 1999 年 [约翰·卡马克](https://zh.wikipedia.org/wiki/約翰·卡馬克) 在《[Quake III Arena](https://en.wikipedia.org/wiki/Quake_III_Arena)》源码中的实现。虽然其原始作者追溯起来可能更早，但这断代码因其包含的“魔数”和注释中的粗口而闻名于世：

```c
float Q_rsqrt( float number )
{
	long i;
	float x2, y;
	const float threehalfs = 1.5F;

	x2 = number * 0.5F;
	y  = number;
	i  = * ( long * ) &y;                       // evil floating point bit level hacking
	i  = 0x5f3759df - ( i >> 1 );               // what the fuck?
	y  = * ( float * ) &i;
	y  = y * ( threehalfs - ( x2 * y * y ) );   // 1st iteration
//	y  = y * ( threehalfs - ( x2 * y * y ) );   // 2nd iteration, this can be removed

	return y;
}
````

**关键步骤解析：**

1.  **`i = * ( long * ) &y;`**：利用指针强制类型转换，将浮点数的位模式（Bit Pattern）直接解释为长整型。这是“Evil Bit Hacking”的核心。
2.  **`i = 0x5f3759df - ( i >> 1 );`**：利用整型运算，通过一个神奇的常数（Magic Number）减去右移一位的值，直接得到 $\frac{1}{\sqrt{x}}$ 的近似解的位模式。
3.  **`y = * ( float * ) &i;`**：将位模式转回浮点数。
4.  **`y = y * ...`**：使用**牛顿迭代法**（Newton's Method）进行一次迭代，将精度提高到满足图形学计算的要求。

## 动机

在计算机图形学（特别是 3D 渲染）中，光照和投影计算涉及大量的矢量归一化（Vector Normalization）操作。归一化公式为 $\hat{v} = \frac{v}{\|v\|}$，其中 $\|v\| = \sqrt{x^2 + y^2 + z^2}$。

这就意味着需要频繁计算 $\frac{1}{\sqrt{x}}$。在早期的硬件上，浮点除法和平方根运算极其昂贵。该算法巧妙地利用位运算绕过了这两个耗时步骤，从而带来巨大的性能优势。

## 分析

### 1\. IEEE 754 Float Representation

要理解这个算法，首先必须理解 **IEEE 754** 单精度（32位）浮点数的存储格式。

  * **符号位 (Sign, $S$)**：1 bit。0 为正，1 为负。对于平方根倒数，我们只考虑正数，故 $S=0$。
  * **指数位 (Exponent, $E$)**：8 bits。存储值为 $0 \sim 255$。引入偏差值（Bias）$B=127$，实际指数为 $E-B$。
  * **尾数位 (Mantissa, $M$)**：23 bits。存储小数部分。由于规范化，隐含了最高位的 1，实际值为 $1 + \frac{M}{2^{23}}$。

令 $L = 2^{23}$，则浮点数对应的数值 $x$ 可表示为：

$$
x = (1 + \frac{M}{L}) \cdot 2^{E-B} \tag{1}
$$

当我们把这段内存直接解释为整数 $I_x$ 时，其数值为：

$$
I_x = E \cdot L + M \tag{2}
$$

### 2\. Aliasing to an Integer as Approximate Logarithm

我们需要计算的是 $y = x^{-1/2}$。两边取对数：

$$
\log_2(y) = -\frac{1}{2} \log_2(x)
$$

对公式 (1) 取对数：

$$
\log_2(x) = \log_2((1 + \frac{M}{L}) \cdot 2^{E-B}) = (E-B) + \log_2(1 + \frac{M}{L})
$$

由于 $\frac{M}{L} \in [0, 1)$，利用线性近似 $\log_2(1+z) \approx z + \sigma$（其中 $\sigma$ 是一个用于修正误差的常数），我们可以得到：

$$
\log_2(x) \approx (E-B) + \frac{M}{L} + \sigma
$$

结合公式 (2) $I_x = E \cdot L + M$，即 $\frac{I_x}{L} = E + \frac{M}{L}$，代入上式：

$$
\log_2(x) \approx \frac{I_x}{L} - B + \sigma \tag{3}
$$

这便是算法的核心数学原理：**浮点数的整数位表示，近似于其自身的对数（线性缩放后）。**

将公式 (3) 代入 $y$ 的对数方程：

$$
\begin{align*}
\frac{I_y}{L} - B + \sigma &\approx -\frac{1}{2} (\frac{I_x}{L} - B + \sigma) \\
I_y &\approx \frac{3}{2}L(B - \sigma) - \frac{1}{2}I_x
\end{align*}
$$

这正好对应了代码中的：

```c
i = 0x5f3759df - ( i >> 1 );
```

其中 $\frac{1}{2}I_x$ 对应位运算 `i >> 1`，而常数 `0x5f3759df` 正是 $\frac{3}{2}L(B - \sigma)$ 的计算结果。

> **关于 $\sigma$ 的补充：**
> 如果 $\sigma = 0$，则近似为 $\log_2(1+z) \approx z$。但这是一条直线连接 (0,0) 和 (1,1)，误差较大。
> 为了最小化误差，我们在 $\log_2(1+z)$ 和 $z+\sigma$ 之间寻找一个最优的 $\sigma$。理论上的最优解（使得误差的均方根最小）可以通过计算积分得到：
> $$
> \\sigma \\approx 0.0430357
> $$
> 实际代码中的 `0x5f3759df` 对应的 $\sigma$ 约为 0.0450466，这是开发者通过实验微调得到的值，甚至比理论值在某些范围内表现更好。

### 3\. Newton's Method

位运算给出了一个非常好的初始猜测值 $y_{est}$，但还不够精确。我们需要使用牛顿法（Newton-Raphson Method）进行迭代。

[Image of Newton's method geometric interpretation on a curve]

我们需要求 $y = \frac{1}{\sqrt{x}}$，即求方程 $f(y) = \frac{1}{y^2} - x = 0$ 的根。
根据牛顿迭代公式 $y_{n+1} = y_n - \frac{f(y_n)}{f'(y_n)}$，代入 $f(y)$ 及其导数 $f'(y) = -2y^{-3}$：

$$
y_{n+1} = y_n - \frac{y_n^{-2} - x}{-2y_n^{-3}} = y_n + \frac{1}{2}y_n(1 - x y_n^2) = y_n(1.5 - 0.5 x y_n^2)
$$

这正是代码的最后一行：
`y = y * ( threehalfs - ( x2 * y * y ) );`

## 总结

虽然现代 CPU 提供了专用的 SSE 指令（如 `rsqrtss`）可以在硬件层面更精确、更快地完成此操作，但“快速平方根倒数”算法依然是软件工程史上最优雅的 hack 之一。它完美展示了数学近似、底层位操作与数值分析的结合。
